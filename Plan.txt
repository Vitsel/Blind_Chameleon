◆◆◆ 문서 중앙화 ◆◆◆
◈기능
┗◇업로드
 ┗▶Client의 파일을 Server에 암호화하여 Upload, Drag & Drop 가능, 업로드 시 접근 권한 설정
 ┗▶다중 Upload 가능
┗◇다운로드
 ┗▶Server의 암호화된 파일을 Client에 Download
 ┗▶다중 Download 가능
┗◇삭제
 ┗▶자신이 소유자인 파일이나 폴더, 소유한 폴더의 하위 파일일 경우 삭제 가능
┗◇이동
 ┗▶삭제와 같은 조건으로 이동 가능, 다른 부서나 개인폴더로 이동 시 접근권한 재설정
 ┗▶개인, 부서, 협업 폴더는 이동 불가능
┗◇폴더생성
 ┗▶개인 또는 속한 부서 폴더 하위에 원하는 폴더 생성
 ┗▶특정 등급 이상은 부서 수준의 협업용 폴더 생성 가능
 ┗▶협업용 폴더 생성시 접근권한 설정
┗◇개인공간
 ┗▶자신만의 공간
┗◇접근권한
 ┗▶모든 파일과 폴더의 접근 권한은 기본적으로 생성자와 해당 위치의 최상위 폴더(부서 수준)의 접근권한을 가짐
 ┗▶협업용 폴더에는 접근 가능한 그룹 다중 선택 및 수정 가능
 ┗▶관리자는 모든 폴더와 파일에 접근 가능
┗◇Default 접근권한
 ┗▶폴더 생성 및 Upload시 기본적으로 설정되는 접근권한 설정 가능 (개인설정창)
┗◇문서 보안
 ┗▶Client와 Server가 연결되어 있을 때만 파일 복호화 키가 주어지고 이를 통해 암호화된 파일을 복호화 가능
 ┗▶연결되지 않을 경우 키를 삭제하고 복호화 불가능
┗◇암호화
 ┗▶pload 하지 않을 Client의 파일 중 원하는 파일을 Server에 Upload할 때와 동일하게 암호화
┗◇복호화
 ┗▶복호화된 파일을 만들지 않고 암호화된 파일 실행 시 복호화된 임시 파일 생성 후 작업이 끝나면 다시 암호화
┗◇List
 ┗▶자신이 접근 가능한 폴더와 파일만 List에 출력
 ┗▶파일명, 업로드 날짜, 업로드한 사원, 파일 크기 표시
┗◇새로고침
 ┗▶일정 주기 또는 버튼 클릭 시 List 새로고침

◈구조
┗◇개인 폴더
 ┗▶개인만 접근 가능한 폴더
┗◇부서 폴더
 ┗▶부서 생성될 때 자동으로 생성되며 해당 부서만 접근 가능한 폴더
┗◇협업용 폴더
 ┗▶권한을 가진 사원에 의해 생성되는 폴더로, 생성시 생성자, 생성자의 부서, 선택된 협업할 부서들이 접근 가능

◈저장위치
┗◇Server의 특정 위치의 Directory



◆◆◆ 장치 제어 ◆◆◆
◈기능
┗◇캠차단
 ┗▶노트북이나 Desktop에 연결된 Cam 사용 금지
┗◇USB 제어
 ┗▶기본적으로 USB 연결이 불가능, Client Program을 통해 관리자에게 요청(시간, 사유 작성)
 ┗▶관리자 승인 시 요청한 시간만큼 USB 연결 가능, 시간이 만료되기 전 연장 신청 가능(시간, 사유 작성)
 ┗▶등록된 특정 USB는 허가 없이 사용 가능
 ┗▶승인이 안된 상태에서 USB 삽입 시 경고창 및 관리자에게 알림           
┗◇마이크 차단
 ┗▶노트북이나 Desktop에 연결된 마이크 사용 금지



◆◆◆ 서버 ◆◆◆
◈기능
┗◇VPN을 이용한 터널링
 ┗▶회사 내부 네트워크는 VPN Server를 통해서만 인터넷 사용 가능
 ┗▶외부 네트워크에서 내부 네트워크로 직접적인 접근 불가
┗◇VPN 접속
 ┗▶회사 외부(자택 등)에서 Client Program을 통해 VPN을 이용해 Server에 접속 가능
 ┗▶Server에 Join 되지 않은 PC는 회사 계정으로 로그인하지 못하므로 이를 감지해 프로그램 실행 시 로그인 요청
 ┗▶로그인 시 VPN 접속
┗◇DB 구축
┗◇웹서버 구축



◆◆◆ 인공지능을 활용한 피싱/유해사이트 차단 ◆◆◆
◈기능
┗◇인공지능
 ┗▶접속하려는 URL이 유해한 사이트인지 판단
 ┗▶이후의 빠른 판단을 위해 차단 목록에 추가
┗◇Client Program
 ┗▶인공지능에 의해 유해한 사이트로 판단되면 접근 차단
 ┗▶관리자에게 차단 목록에서 제외 요청 팝업 (사유 작성)
┗◇차단목록
 ┗▶유해한 사이트의 URL을 저장해 이후의 판단에 사용
┗◇제외목록
 ┗▶인공지능이 유해하다고 판단한 URL 중 유해하지 않은 사이트의 URL을 저장해 이후의 판단에 사용



◆◆◆ 채팅 ◆◆◆
◈기능
┗◇UI
 ┗▶친구 목록, 채팅방 목록 등의 Tab 존재
 ┗▶친구 목록에는 부서, 이름, 계급과 상태(로그인, 로그아웃, 자리비움) 표시
┗◇친구 추가
 ┗▶사원ID(로그인 ID가 아닌 정수로 이루어진 ID)를 이용해 친구 요청 및 수락
┗◇채팅방 생성 및 초대
 ┗▶채팅방 생성(채팅방 이름 설정) 후 친구 목록에서 채팅방에 초대
 ┗▶초대 내용에는 채팅방 이름과 채팅방 개설자, 초대 메시지 기제
 ┗▶초대시 초대 받은 사원에게 알림 (카톡 왔을때 처럼 오른쪽 아래에 작은 팝업)
 ┗▶초대된 사원은 수락 시 채팅방에 합류
┗◇채팅방 목록
 ┗▶카카오톡 채팅방 목록과 동일
┗◇채팅
 ┗▶카카오톡과 비슷한 UI
 ┗▶자신이 속한 채팅방에서 채팅이 올라오면 알림(카카오톡과 동일)
 ┗▶표시되는 이름은 부서, 이름, 계급
 ┗▶채팅방 접속자 목록(상태 표시)
 ┗▶메시지별 읽지 않은 사람 표시(카카오톡의 1)
┗◇파일송수신
 ┗▶개발 중 필요하다면 파일 형식, 크기 제한
 ┗▶사진의 경우 체팅창에 바로 보이기
 ┗▶사진 클릭 시 크게 보기
 ┗▶송수신한 파일은 모두 암호화
 ┗▶파일 다운로드 시 문서중앙화와 같이 문서보안 적용



◆◆◆ 웹 ◆◆◆
◈기능
┗◇로그인
 ┗▶관리자 계정으로 로그인
┗◇계정 생성
 ┗▶로그인ID, 비밀번호, 성별, 소속부서, 계급, 소속그룹 등의 계정 정보 작성
 ┗▶최초비밀번호는 특정 길이의 영문 소대문자, 특수문자, 숫자를 포함한 난수
 ┗▶비밀번호를 제외한 계정정보와 자동으로 증가하는 사원ID(로그인 ID와 다른 정수)는 DB에 저장
┗◇계정 관리
 ┗▶트리형식의 부서별 직원 목록
 ┗▶특정 직원 클릭 시 직원 정보 변경 가능
 ┗▶ListBox, RadioButton 등 활용, 단순 입력창 최소화
 ┗▶부서 변경시 즉시 계정 목록에서 해당 직원의 부서 이동
 ┗▶계정 정보에는 기본적인 정보들과 장치관리에서 제한중인 각 장치의 상태(제한, 허용) 표시
┗◇문서관리
 ┗▶Server에 저장된 문서중앙화에서 사용하는 Root Directory 이하의 모든 폴더가 Tree 형식으로 표시
 ┗▶각 폴더를 선택하면 우측에 파일 목록 표시
 ┗▶각 파일은 파일명, 업로드 날짜, 업로드한 사원, 파일 크기 표시
 ┗▶모든 폴더에 접근할 수 있으며, 모든 파일 업로드, 다운로드, 삭제 가능
 ┗▶모든 위치에 폴더 생성 가능
┗◇로그 출력
 ┗▶웹에서는 로그 확인만 수행, 로그 기록은 Client나 Server Program이 수행
 ┗▶DB에 저장된 로그를 알기 쉽게 표시
 ┗▶시간, 종류 등 기준으로 정렬 가능
 ┗▶시간, 종류, 내용 등 검색 가능
┗◇USB 접근 승인
┗◇차단목록, 제외목록 관리
 ┗▶차단목록과 제외목록의 추가, 삭제, 수정 기능 제공
┗◇차단건의 확인 및 승인
